### Key Points

1. **One MSAL Instance**: Shared across all components (singleton)
2. **Multiple State Instances**: Each `useAuth()` call has its own state
3. **Event-Based Sync**: MSAL events keep all states synchronized
4. **Automatic Updates**: No manual coordination needed between components

### Code Changes

**Added to `useAuth.ts`:**

```typescript
// Listen for MSAL authentication events to sync state across all hook instances
useEffect(() => {
  const callbackId = msalInstance.addEventCallback((event: EventMessage) => {
    if (event.eventType === EventType.LOGIN_SUCCESS && event.payload) {
      const payload = event.payload as AuthenticationResult;
      if (payload.account) {
        updateAuthState(payload.account, payload.accessToken, payload.idToken || '');
        setJustLoggedIn(true);
      }
    }
    // ... handle other events
  });

  return () => {
    if (callbackId) {
      msalInstance.removeEventCallback(callbackId);
    }
  };
}, [updateAuthState]);
```
